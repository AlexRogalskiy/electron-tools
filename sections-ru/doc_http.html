Список HTTP кодов состояния

Когда пользователь посещает веб-страницу, браузер клиента делает запрос на сервер, на котором находится веб-страница. Прежде чем браузер получит и отобразит веб-страницу, сервер, на котором находится веб-страница, вернет заголовок сервера, содержащий код состояния HTTP, чтобы ответить на запрос браузера.

Коды состояния на русском языке

Общие HTTP коды состояния
200 - OK (хорошо)
301 - Moved Permanently (перемещено навсегда)
404 - Not Found (не найдено)
500 - Internal Server Error (внутренняя ошибка сервера)

HTTP код состояния состоит из трех десятичных цифр. Первая десятичная цифра определяет тип кода состояния, а последние две цифры не имеют никакой роли для классификации. HTTP коды состояния делятся на 5 типов:

Классификация статус-кодов HTTP

Классификация Описание
1** Информационное. Запрос принят, процесс продолжается.
2** Успех. Сообщение было успешно принято.
3** Перенаправление. Для завершения запроса необходимо произвести ряд действий.
4** Ошибка клиента. Запрос содержит синтаксическую ошибку и не может быть удовлетворен.
5** Ошибка сервера. Сервер не может выполнить внешний корректный запрос.

Список кодов состояния HTTP

Код Код состояния Описание
100 Continue Продолжить. Клиент должен продолжить запрос.
101 Switching Protocols Протокол переключения. Сервер переключает протокол в соответствии с запросом клиента. Можно только переключиться на более сложные протоколы, например, переключиться на новую версию HTTP-протокола.
200 OK Запрос был успешным. Обычно используется для запросов GET и POST.
201 Created Она была создана. Успешно запросил и создал новый ресурс.
202 Accepted Принято. Принятый запрос, но не обработан.
203 Non-Authoritative Information Неавторизованная информация. Запрос был успешным. Но возвращаемая метаинформация не является исходным сервером, а копией.
204 No Content Нет содержания. Сервер успешно обработал, но не вернул содержимое. Убедитесь, что браузер продолжает отображать текущий документ без обновления страницы.
205 Reset Content Сбросить содержимое. Процесс сервера выполняется успешно, и пользовательский терминал (например, браузер) должен сбросить вид документа. Очистить поля формы браузера с этим кодом возврата.
206 Partial Content Некоторое содержание. Сервер успешно обработал некоторые GET запросы.
300 Multiple Choices Несколько вариантов. Запрошенный ресурс может включать несколько местоположений, и соответствующий список функций и адресов ресурсов может быть возвращен для выбора пользовательского терминала (например, браузера).
301 Moved Permanently Перемещение навсегда. Запрошенный ресурс постоянно перемещается в новый URI, возвращаемая информация будет включать новый URI, и браузер автоматически будет перенаправлен на новый URI. Любой новый запрос в будущем должен использовать новый URI вместо
302 Found Перемещение временно. Аналогично 301. Но ресурсы временно перемещаются. Клиент должен продолжать использовать исходный URI.
303 See Other См. Другие адреса. Аналогично 301. Просмотр с использованием запросов GET и POST.
304 Not Modified Не изменен. Запрошенный ресурс не был изменен. Когда сервер возвращает этот код состояния, ресурсы не возвращаются. Клиент обычно кэширует доступные ресурсы, предоставляя заголовок, указывающий, что клиент хочет вернуть только ресурсы, измененные после указанной даты.
305 Use Proxy Используйте прокси. Запрошенный ресурс должен быть доступен через прокси-сервер.
306 Unused Код статуса HTTP, который устарел.
307 Temporary Redirect Временная переадресация. Аналогично 302. Использовать перенаправление запроса GET.
400 Bad Request Синтаксис запроса клиента неверен, и сервер не может понять его.
401 Unauthorized Запрос на аутентификацию пользователя.
402 Payment Required Зарезервировано для будущего использования.
403 Forbidden Сервер понимает запрос от клиента, но отказывается выполнить запрос.
404 Not Found Сервер не смог найти ресурс (веб-страницу) на основе запроса клиента.
405 Method Not Allowed Метод в запросе клиента запрещен.
406 Not Acceptable Сервер не может выполнить запрос на основе свойств содержимого запроса клиента.
407 Proxy Authentication Required Запрос требует аутентификации прокси-сервера, аналогичного 401, но запрашивающий должен использовать прокси-сервер для авторизации.
408 Request Time-out Сервер ожидает, что клиент отправит запрос, который занимает слишком много времени для тайм-аута.
409 Conflict Сервер может завершить запрос PUT клиента и вернуть этот код, и сервер столкнулся с конфликтом при обработке запроса.
410 Gone Ресурс, запрошенный клиентом, больше не существует. 410 отличается от 404. Если ресурс ранее был удален навсегда и используется код 410, разработчик сайта может указать новое местоположение ресурса через код 301.
411 Length Required Сервер не может обрабатывать запросы, отправленные клиентом без Content-Length.
412 Precondition Failed Предварительные требования для информации запроса клиента неверны.
413 Request Entity Too Large Поскольку запрошенный объект слишком велик, сервер не может его обработать, поэтому запрос отклоняется. Чтобы предотвратить непрерывные клиентские запросы, сервер может закрыть соединение. Если только сервер не может обработать его временно, он будет содержать ответное сообщение Retry-After (Повторите попытку).
414 Request-URI Too Large Запрошенный URI слишком длинный (URI обычно является URL-адресом), и сервер не может его обработать.
415 Unsupported Media Type Сервер не смог обработать формат носителя, который пришел с запросом.
416 Requested range not satisfiable Недопустимый диапазон клиентских запросов.
417 Expectation Failed Сервер не может встретить заголовок запроса Expect.
500 Internal Server Error Ошибка внутреннего сервера, не удалось выполнить запрос.
501 Not Implemented Сервер не поддерживает запрошенную функцию и не может выполнить запрос.
502 Bad Gateway Сервер, действующий как шлюз или прокси, получает неверный запрос с удаленного сервера.
503 Service Unavailable Из-за перегрузки или обслуживания системы сервер временно не может обрабатывать запрос клиента. Длина задержки может быть включена в заголовок сервера Retry-After (Повторите попытку).
504 Gateway Time-out Сервер, действующий как шлюз или прокси, не получил запрос от удаленного сервера вовремя.
505 HTTP Version not supported Сервер не поддерживает запрошенную версию протокола HTTP и не может завершить обработку.
